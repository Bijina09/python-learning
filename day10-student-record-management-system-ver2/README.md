# Day 10 - Student Record Management System (Version 2)

## Project

A console-based Student Record Management System for managing student records, tracking marks, and generating class reports.

## Features

- Add new students
- Prevent duplicate Student IDs
- View all students
- Search students by Student ID
- Update student details
- Delete students
- View class statistics (highest, lowest, and average marks)
- Generate pass/fail reports
- Menu-driven interface

## Concepts Practiced

- Functions
- Lists of dictionaries
- CRUD operations (Create, Read, Update, Delete)
- Helper functions for code reuse
- Dictionary initialization using literals
- Case-insensitive string comparison using `lower()`
- `enumerate()` with `start`
- Conditional statements
- Accumulator variables for calculating totals
- Finding maximum and minimum values

## What I Learned

- Reusing the structure of a previous project makes developing similar applications much faster.
- A unique Student ID simplifies searching, updating, and deleting records.
- Maximum, minimum, and average values can be calculated efficiently by iterating through a list once.
- Organizing code into small functions makes the program easier to read, test, and maintain.
- Similar program structures can be adapted to solve different real-world problems with only a few changes.

## Future Improvements

- Validate numeric input for age and marks.
- Create a helper function to find students by Student ID.
- Save and load student records from a file.
- Display messages when there are no passing or failing students.
- Sort students by name or marks.
