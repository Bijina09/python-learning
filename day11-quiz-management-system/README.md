# Day 11 - Quiz Management System

## Project

A console-based Quiz Management System for managing quiz questions and allowing users to take a quiz while tracking their score.

## Features

- Add new quiz questions
- Prevent duplicate questions
- View all questions
- Delete questions
- Start the quiz
- Validate answers (A, B, C, or D)
- Display the final score
- Show a performance message based on the score
- Menu-driven interface

## Concepts Practiced

- Functions
- Lists of dictionaries
- Helper functions for code reuse
- Dictionary initialization using literals
- Case-insensitive string comparison using `lower()`
- `enumerate()` with `start`
- Conditional statements
- Loops and iteration
- Input validation
- Accumulator variables for calculating scores

## What I Learned

- A program can both manage data and use that data to perform another task, such as conducting a quiz.
- Helper functions make repeated display logic cleaner and easier to maintain.
- Input validation improves the user experience by preventing invalid answers.
- Accumulator variables can be used to keep track of scores during a quiz.
- Breaking the program into small, focused functions makes the overall code easier to read and organize.

## Future Improvements

- Update existing quiz questions.
- Randomize the order of quiz questions.
- Shuffle the answer options for each question.
- Save and load quiz questions from a file.
- Display the user's percentage score along with the final result.
