# Day 9 - Inventory Management System

## Project

A console-based Inventory Management System for managing products, tracking stock levels, and calculating the total inventory value.

## Features

- Add new products
- Prevent duplicate Product IDs
- View all products
- Search products by Product ID
- Update product details
- Delete products
- View low stock products
- Calculate total inventory value
- Menu-driven interface

## Concepts Practiced

- Functions
- Lists of dictionaries
- CRUD operations (Create, Read, Update, Delete)
- Helper functions for code reuse
- Case-insensitive string comparison using `lower()`
- Dictionary initialization using literals
- `enumerate()` with `start`
- Conditional statements
- Accumulator variables for calculating totals

## What I Learned

- Using a unique Product ID makes it easier to search, update, and delete products.
- Helper functions reduce repeated code and make programs easier to maintain.
- Accumulator variables can be used to calculate totals while iterating through a list.
- Filtering data with conditions makes it easy to generate reports such as low stock products.
- Breaking a program into small functions makes the code easier to read and organize.

## Future Improvements

- Validate numeric input for price and quantity.
- Create a helper function to find products by Product ID.
- Save and load inventory data from a file.
- Add product categories and supplier information.
- Sort products by name, price, or quantity.
