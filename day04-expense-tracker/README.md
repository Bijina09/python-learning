# Day 4 - Expense Tracker

## Project

A console-based expense tracker that allows users to record and manage daily expenses.

## Features

- Add new expenses
- View all expenses
- Search expenses by category
- Calculate total spending
- Display the highest expense(s)
- Menu-driven interface

## Concepts Practiced

- Functions
- Lists and dictionaries
- Loops (`for`, `while`)
- Conditional statements (`if`)
- `return`
- `break`
- `match-case`
- `enumerate()`
- Searching through a list
- Aggregating values using loops
- Finding the maximum value in a collection
- Case-insensitive string comparison using `lower()`

## What I Learned

- How to process a collection of data to calculate totals and find maximum values.
- The difference between modifying an object and reassigning a variable to a new object.
- How references work in Python and why lists store references to objects rather than variables.
- How to handle edge cases such as an empty list or multiple items sharing the highest value.
- The importance of choosing descriptive variable names like `expenses` and `current_expense`.

## Future Improvements

- Validate numeric input using `try`/`except`.
- Edit existing expenses.
- Delete an expense.
- Display expenses by date.
- Store expenses in a file so they persist after closing the program.
- Show spending summaries by category.
