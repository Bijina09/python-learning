    # Day 8 - Library Management System (Version 2)

    ## Project

    A console-based Library Management System for managing books and tracking their borrowing status.

    ## Features

    - Add new books
    - Prevent duplicate books (same title and author)
    - View all books
    - Search books by title
    - Borrow books
    - Return books
    - View borrowed books
    - View available books
    - Delete books
    - Menu-driven interface

    ## Concepts Practiced

    - Functions
    - Lists of dictionaries
    - Boolean values (`True` / `False`)
    - Helper functions for code reuse
    - Default function parameters (`None`)
    - Dictionary initialization using literals
    - Case-insensitive string comparison using `lower()`
    - `enumerate()` with `start`
    - Custom counters for filtered output

    ## What I Learned

    - `None` is commonly used as a default parameter to represent "no value provided."
    - `is not None` checks whether a value was supplied, while `not value` checks whether a value is considered false.
    - Helper functions can make repeated display logic much cleaner and easier to maintain.
    - Reducing duplicated responsibilities is more important than simply reducing the number of lines of code.
    - Small logic mistakes are harder to catch than syntax errors, so reading conditions carefully is important.

    ## Future Improvements

    - Save and load books from a file.
    - Search by author.
    - Allow multiple copies of the same book.
    - Store the borrower's name and borrowing date.
    - Add categories and ISBN numbers.
